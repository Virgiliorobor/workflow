The 6-layer ICM system

L0
CLAUDE.md — Always loaded. The map. Identity, routing table, naming conventions. ~800 tokens.
L1
CONTEXT.md — Loaded on workspace entry. Stage map, how stages connect, routing logic.
L2
Stage contracts — One per stage. Loaded per task. Inputs / Process / Output / Done looks like.
L3
_config/ files — Loaded selectively. Voice, format patterns, constraints, business rules. Stable across runs.
L4
output/ directories — Working files. Stage N's output is Stage N+1's input. Human review at each handoff.
L5
skill-starters/ — Optional prompt templates. Reusable “skills” to start work in each stage quickly and consistently.
